跨级传

provide
跨级传递 provide vs inject
顶级组件 → 底层组件；中间可以经过多层组件
仅允许上向下传递|流转，不允许底层组件向顶级组件传递|反向流转
采用发布-订阅|提供-插入的模式
顶层组件引入 provide包、底层组件引入 inject包
可以传递数据；还可以传递方法
不可以在某个事件处理函数中使用：provide() can only be used inside setup()
传递数据
可以使用多个provide和inject
可以传递普通静态数据，也可以传递响应式数据
可以传递简单数据，也可以传递对象
使用延时查看响应数据的变化
顶级组件使用provide提供数据
import {provide} from 'vue'

provide('key0', data)
provide('key1', dataRef.value)
底层组件使用inject接收数据
import {inject} from 'vue'

let para = inject('key0')
let para = inject('key1')
传递方法
底层组件可以使用顶层组件的方法修改顶层组件的数据；虽然有悖于数据流的单向原则，但是仍然遵循了谁的数据谁修改的原则
顶级组件使用provide提供已经定义好的方法doGreeting
用你提供的方法修改你的数据
function doGreeting() {
    refDate.value++
  }
provide('doGreetingFn', doGreeting)
底层组件使用inject接收传递的方法，并在事件处理函数fn中实现；fn是普通点击事件
doGreet接收传递来的方法，要执行，必须使用()
let doGreet = inject('doGreetingFn')
//直接执行
doGreet()
//在方法中执行
function fn() {
  doGreet()
}
或直接在结构中使用传递的方法
<div @click="doGreet">直接使用 fn</div>
全局使用axios
入口文件引入定义main.js
import axios from 'axios'

app.provide('axios',axios)
组件中使用
let axios = inject('axios')
