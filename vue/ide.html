项目开发
使用Vue 官方的项目脚手架工具Vite创建项目，根据提示操作即可；详细情况，看查看Vite 官方文档
基本过程
创建空白文件夹或选择你的项目开发总文件夹；不能出现中文符号；不要在桌面创建项目
进入文件夹，打开命令行窗口CMD，输入以下指令，创建项目并指定项目名；字母开头
选择Vue框架 - 使用方向键选择，回车确认
定制Vue框架 - 只配置基本的功能，typescript、语法检查等功能暂且不选；后期使用路由管理vue-router和状态管理pinia
根据提示逐步执行：进入项目目录、安装依赖并运行项目

注意：项目创建完毕后，后续操作请使用Vs Code完成

按住CTRL + 单击项目链接，即可打开Vue + Vite的欢迎界面

项目解读
node_modules
依赖包
生产环境依赖 dependencies：项目上线后，所必须的依赖，如网络请求axios
开发环境依赖 devDependencies：开发过程中需要的依赖，发布上线后，不再需要，如CSS预处理
dist
项目发布后才有
将其推送到服务器上，项目就进入生产环境的测试环节；经过一系列的调试、重新发布、再上线，项目进入运维环节
public
公共资源，可以直接被项目使用，如图片，不需要解析
项目发布后，这里的资源会 原封不动 的拷贝到发布文件夹dist
src
开发过程操作最多的目录，您大展身手的舞台
包括重要的两个文件：
1. main.js
逻辑入口文件；引用并挂载App.vue；引入样式库和第三方库等
import './assets/main.css'
import 'font-awesome/css/font-awesome.min.css'
2. App.vue
主入口组件
特别的 单文件组件 SFC，包括逻辑、结构和样式
.逻辑中可以引入其它组件、库和模块
import { ref } from 'vue';
import axios from 'axios';
.样式中可以其它样式
@import url(//at.alicdn.com/t/c/font_4090429_8xfmdtpelxu.css);
index.html
项目的主页文件
指定了挂载点和挂载的组件
<div id="app"></div>
<script type="module" src="/src/main.js"></script>
package.json
依赖包管理文件；定制命令
区分生产环节依赖 dependencies和开发环境依赖 devDependencies；更多信息，请访问Node
由于包的更新和大小等因素，项目分享或提交到代码仓库时，这部分会被忽略。用户拿到代码，第一件事情是安装依赖；否则项目不能运行
{
    "name": "sfc",
    "version": "0.0.0",
    "private": true,
    "type": "module",
    "scripts": {
        "dev": "vite",

可以指定主机地址，如0.0.0.0，表示局域网地址
"dev": "vite --host 0.0.0.0",
vite.config.js
vite的配置，如server的配置：项目打开、端口、代理等等；默认没有，需手动添加
网络请求出现跨域时，必须要做代理 Proxy
更多配置，请访问官方文档 - 配置 Vite
server: {
    open: true,         //启动项目自动弹出浏览器；默认false，手动启动
    host: '0.0.0.0',    //指定主机；默认是locahost
    port: '5173',       //默认端口5173
    proxy: {
        '/food': {
        target: 'http://127.0.0.1:3000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')   //不可以省略rewrite
        }
    }
}
