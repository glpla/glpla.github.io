计算属性

Computed
引入
数据绑定时，可以使用简单的表达式实现数据的2次处理；如根据age判断DOM的渲染
逻辑复杂时，应该使用计算属性实现
优点
需要引入 computed 包
计算属性是响应式；数据响应时，不会刷新页面
计算属性有返回值，且仅在其响应式依赖更新时才重新计算
而，方法可以没有返回，且每次执行方法都会导致页面渲染
计算属性是一个属性，不是方法，使用时不需要()
执行2次方法inc和执行2次计算属性dec
创建方法inc和计算属性dec
import { computed } from 'vue';
const inc = () => {
  console.log('do function inc');
}
const dec = computed(() => {
  console.log('do computed dec');
})
使用2次方法；2次计算属性
<div>第1次 inc - {{ inc() }}</div>
<div>第2次 inc - {{ inc() }}</div>
<div>第1次 dec - {{ dec }}</div>
<div>第2次 dec - {{ dec }}</div>
分析
数据变化时，2次方法都会执行
数据变化时，计算属性仅第1次执行；第2次没有检测到数据变化，就不会执行

do function inc
do function dec
使用value指定属性值
import { ref, computed } from 'vue';
let age = ref(18)
const res = computed(() => {
    return age.value >= 18 ? 'adault' : 'teenage'
})
<div> {{ res }} </div>
参照B站规则处理关注、粉丝、点赞和播放数量
传递参数
计算属性是一个属性，不是方法，所以不能直接传递参数
通过闭包的形式间接传递参数
将资源服务器地址和图片路径拼接为一个完整的在线地址
const getUrl = computed(() => {
    return (url) => {
        return 'http://127.0.0.1:3000' + url
    }
})
根据成绩求绩点
let score = ref(80)
const grade = computed(() => {
  return (score) => {
    switch (Math.floor(score / 10)) {
      case 6:
        return 'D'
      case 7:
        return 'C'
      case 8:
        return 'B'
      case 9:
        return 'A'
      default:
        return 'E'
    }
  }
})
以上逻辑有什么需要改进的地方
<div>grade {{ score }} - {{ grade(score) }}</div>
