SFC
单文件组件 - Single File Component，每个文件都是一个组件
文件名采用大驼峰，后缀名为.vue；如HomeView.vue
将一个 Vue 组件的模板、逻辑与样式封装在单个文件；遵循：结构 - 样式 - 逻辑"三分离"原则
<template>
    //结构
</template>

<script>
    //逻辑
</script>

<style>
    //样式
</style>
结构和样式同vue2相比，没有什么变化
逻辑除了选项式API，还提供了组合式API
1. 选项式 Options API
<script>
    export default {
        data() {
            return {
            }
        },
        methods: {
        },
        mounted() {
        }
    }
</script>
2. 组合式 Composition API - 按需导入，并采用setup语法糖；我们采用组合式 Composition API
<script setup>
    
</script>
经常用到的包有：
import { ref } from 'vue'
import { reactive } from 'vue'
import { computed } from 'vue'
import { watch } from 'vue'
import { onMounted } from 'vue'
定制开发模板
Vs Code的扩展 Vue VSCode Snippets 提供了很多内置的代码片段
在新组件中，输入vb或v3后，选择vbase-3-setup可以快速生成模板，默认采用带根节点的结构<template>、逻辑<script>在结构<template>后面；样式<style>CSS预处理器scss
我们调整为：逻辑在最前面、去掉根节点、不带CSS预处理
1. 打开用户目录 → .vscode → 扩展extension → vue-vscode-snippets → snippets → vue.json，定位到vbase-3-setup
2. 删除结构<template>中的根节点<div>
3. 移动逻辑<script>到最上面
4. 保存并重启 Vs Code，扩展生效
"Vue Single File Component Setup Composition API": {
    "prefix": "vbase-3-setup",
    "body": [
        "<script setup>",
        "",
        "</script>",
        "",
        "<template>",
        "",
        "</template>",
        "",
        "<style scoped>",
        "",
        "</style>"
    ],
    "description": "Base for Vue File Setup Composition API with SCSS"
}
现在模板为：
<script setup>
//逻辑
</script>
                
<template>
    //结构
</template>

<style scoped>
    //样式
</style>
