@Component
SFC - Single File Component，单文件组件
vue中每个组件都是以SFC的形式存在，后缀名是.vue
包括：结构<template>、逻辑<script>和样式<style>，其中，结构<template>是必须的
每个组件可以被其它组件导入|引入并多次使用
适合模块化开发和团队协作
基本过程
创建子组件，后缀名为.vue，如 Demo.vue
<script setup>
    //逻辑
</script>
    
<template>
    //结构
</template>

<style>
    //样式
</style>
父组件在<script>引入子组件

注意路径关系；可以使用相对路径；也可以使用路径别名@从src开始引用

不能省略后缀名.vue

引入时，输入组件部分或全名，IDE会自动获取组件及补充相对路径；注意看代码提示

<script setup>
    import Demo from './Demo.vue'
</script>

为了按需引入，减少加载时间，特别是首屏加载，通常采用 异步组件 的形式引入；

采用了异步组件，打包的时，会拆分成多个.js文件

import { defineAsyncComponent } from 'vue'

const AsyncComp = defineAsyncComponent(() =>
  import('./components/MyComponent.vue')
)
父组件在结构<template>中使用子组件

可以使用单标签或双标签

可以多次使用

<template>
    <h1>Here is a child component!</h1>
    <Demo />
    <Demo></Demo>
</template>
全局组件
main.js中引入并声明
其它组件直接使用，无需引入
import Com from './components/Com.vue';

app.component('Com', Com)
