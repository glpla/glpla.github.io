编程路由

@ Router
除了使用 <router-link> 创建路由|声明式路由，还可以借助 router 相关 API|编程路由实现；类似于 <a> 和 location.href 的关系
需引入路由包 useRouter 并创建路由实例router
可以使用 普通路由|路径path；也可以使用 命名路由|name
还可以指定路由参数 params 或 query
useRoute() 是一个全局注入的函数，用于在组件中获取当前激活的路由信息。它返回一个 RouteLocationNormalized 对象，包含了当前路由的所有属性，如 path、params、query 等
useRouter() 同样是全局注入的函数，用于获取路由器实例，从而可以进行导航操作，如 push、replace、go 等
仅在 Vue 3 的 Composition API 环境下使用
更多信息，请访问 官方文档 - useRouter
import { useRouter } from 'vue-router'
const router = useRouter()
编程式路由常用方法
分类 说明
push 导航时向 history 添加新记录|压入路由，并跳转到目标；单击<router-link>时触发；可以逐级返回
replace 导航时不会向 history 添加新记录，替换|取代了当前的条目；不可返回
go(n) 导航到历史记录中的特定位置；可以前进或后退；参数 n 是一个整数，相对于当前位置的偏移量
由于属性 to 与 router.push 接受的对象种类相同，所以两者的规则完全相同
// 查询路由
router.push('/user?id=100')

// 查询路由，结果是 /register?plan=private
router.push({ path: '/register', query: { plan: 'private' } })

// 动态路由 - 静态和动态
router.push('/user/eduardo')
router.push({ path: '/detail/' + ind })

// 动态路由，并加上参数，让路由建立 url；只能使用命令路由name
router.push({ name: 'user', params: { username: 'eduardo' } })

//禁止返回路由
router.push({ path: '/user', query: { bread: 'user' }, replace: true })
router.replace({ path: '/user', query: { bread: 'user' } })
3秒后跳转至首页 - 延时实现 setTimeout
在生命周期函数中实现路由导航
//引入
import { onMounted } from 'vue'
import { useRouter } from 'vue-router'

//使用 
const router = useRouter()
onMounted(()=>{
    setTimeout(()=>{
        //router.push('/home')
        router.replace('/home') //这里通常使用replace模式
    }, 3000)
})
