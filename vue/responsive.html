为什么使用响应式数据？
Vue 会自动检测响应式数据的变化，进而更新 DOM - 数据驱动 Data Driven
更多响应式原理，请访问 响应式基础、深入响应式系统
ref
创建基本数据类型、复合数据类型的响应式数据
ref() 将传入参数的值包装为一个带 .value 属性的 ref 对象
ref 的 .value 属性是响应式
1. 引入包 ref - 下述各例略
import { ref } from 'vue'
2. 声明响应式变量 - 执行函数、传入参数、变量接收
const count = ref(0)
3.1 结构中使用 - 直接使用，vue会自动获取其value
<div>{{count}}</div>
3.2 逻辑中使用 - 赋值时需额外指定value属性，其它情况直接使用，如监听；可以由开发环境的插件自动补齐value属性
console.log(count)
console.log(count.value)
控制台输出，查看响应式变量的细节，如value属性在哪里
reactive
只能创建复合数据类型的响应式数据，如对象、数组
不用指定value属性
1. 引入包 reactive - 下述各例略
import { reactive } from 'vue'
2. 声明响应式变量 - 执行函数、传入参数、变量接收
const obj = reactive({
    id: 10,
    name: 'gz'
})
不能 声明简单数据类型 - value cannot be made reactive: 10
const count = reactive(10)
3.1. 结构中使用
<div>{{obj.id}} - {{obj.name}}</div>
3.2. 逻辑中使用 - 只能单个属性赋值
const done = () => {
    obj.id = 20
}
console.log(obj)
控制台输出，查看响应式变量的细节
不能 重新赋值|分配对象，否则就失去响应式
const done = () => {
    obj = {
      id: 20,
      name: 'gz'
    }
}
解决方案1：可以使用对象的分配方法，更新数据
const done = () => {
    Object.assign(obj,{
        id: 20,
        name: 'gz'
    })
}
解决方案2：可以使用对象的分配方法，更新数据
经过函数ref()、reactive()处理后，普通的数据或对象就变成了响应式
使用log查看一个变量是否具有响应式数据：如果带有Proxy/refImpl，是响应式数据；否则就是普通变量
实际开发中，更多使用ref；因为可以直接改：管你是简单数据还是复合数据
. 简单数据，当然只能ref()
. 如果需要简单的对象，ref()、reactive()都可以
. 层级比较深的对象，请使用reactive()
[]练习

使用reactive()声明嵌套复合数据类型并验证其响应式
