浮动 Float
浮动元素会脱离文档流document flow；
元素在文档中出现的顺序不一样，最后渲染的布局结果也不一样；
必须要清除浮动[你不清试试]；
现在：仅仅在一些特殊应用如：图文混排或首字下沉、根据形状浮动填充时，才会使用；
其它布局请采用弹性盒子flex或网格grid；
清除浮动
清除的方法很多。如：为父元素指定高度；父元素溢出隐藏；
为了不增加额外的标签，通常以伪元素::after的形式为浮动元素的父元素指定清除类；
    .clear::after {
        content: '';
        display: block;
        clear: both;
    }

定位 Position
相对定位Relative/绝对定位Absolute
相对定位：不脱离文档流；参考对象是自己；很少单独使用，通常为子元素提供定位依据，即：父相子绝
绝对定位：脱离文档流；参考对象是显示的设置了定位属性(相对或绝对都可以)的最近的父元素；如果父元素没有指定定位属性，则一级一级往上寻找，直至<body>；绝对定位元素默认会待在原来的位置，可以利用关键字top right bottom left和负边距定位元素。

[分析] 利用float虽然可以实现，但是不符合页面SEO：最重要中间是主体部分，应该先显示，而侧边栏的应该稍后；所以文档流中main在前，侧边栏在后；抛弃float，使用定位实现；

    }
    .pos-l {//子绝
        position: absolute;
        left: 0;
        top: 0;
        width: 70px;
        //...
    }

    .pos-r {//子绝
        position: absolute;
        right: 0;
        top: 0;
        width: 70px;
        //...
    }

固定定位Fixed
固定定位元素以视口viewport为参考，固定在屏幕上，不会随页面的滚动而移动。通常用来定制导航，如顶部导航、侧边导航和底部导航等。
[案例] 固定在版心两侧的导航
[思路] 贴近右侧：左偏移浏览器的一半（50%），再增加半个版心的margin-left；贴近左侧同理；导航通常的解决方案是：当前页面位于顶端时，暂时隐藏侧边导航条；当页面向下滚动到一定位置时，再显式侧边导航栏；一般需要借助JS实现。
[案例] 后端管理页面
[思路] 利用固定定位属性top、right、bottom、left分别为0，进而占满整个视口；
粘性定位
兼有相对定位和绝对定位的特性；以视口viewport为参考(具有固定定位特性)，占有原来位置(具有相对定位特性)；必须指定left、top、right、bottom某一个；适合banner/swiper+主导航+内容的网页。网页滚动时候，banner移出，导航保留/黏贴在顶端，内容也逐步移出；还可以使用JS实现黏性效果。
注意
1.其父元素不能设置overflow:hidden
2.浏览器支持不是很广泛；更多信息请查看https://caniuse.com/
3.以上案例，可以参考示例；
