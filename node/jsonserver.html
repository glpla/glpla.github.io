json-server

JSONServer
快速搭建HTTP服务；需要node.js环境支持
为应用提供JSON数据服务 - 仅仅是针对json文件的解析和应用
更多信息，请访问NPM Json-Server；跟随文档操作即可；更多使用，请访问从入门到膨胀
方法
GET - 数据查询；配合id查询特定记录
POST - 新增数据；会自动增加id
PUT|覆盖 和 PATCH|更新 - 修改数据
DELETE - 删除数据
创建服务器
创建空白项目文件夹作为服务器，如server
项目初始化 - 完成后创建package.json包管理配置文件；参数-y表述默认创建
npm init -y
安装json-server依赖 - 通常独立全局-g安装，可以监听对于任何位置的json数据的请求 - 指哪打哪
npm install json-server -g
创建数据文件 - students.json；包括用户id，姓名和星数；初始应为空，通过添加和修改逐步增加
{
  "digital": [
    {
      "uid": 20220110,
      "uname": "gl",
      "rank": 0
    }
  ],
  "software": [
    {
      "uid": 20220210,
      "uname": "gl",
      "rank": 0
    }
  ]
}
在CMD控制台启动服务 -监听数据文件 students.json；系统自动检测并给出接口的地址；注意当前打开的CMD位置和监听数据文件的路径关系

默认端口3000

json-server students.json

指定端口 - 和项目端口保持一致，避免浏览器跨域

json-server students.json --port=5173

GET - 请求数据；挂载完毕
onMounted(() => {
  axios.get('http://127.0.0.1:5173/software')
    .then(res => {
      //赋值
    })
})

可以封装函数

onMounted(() => {
  loadData()
})
func loadData(){
  axios.get('http://127.0.0.1:5173/software')
    .then(res => {
      //赋值
  })
}

POST - 增加数据；提交用户id，姓名和星数；完毕后重新拉取数据
function add() {
  axios.post('http://127.0.0.1:5173/software', {
    uid: uid.value,
    uname: uname.value,
    rank: rank.value
  })
    .then(res => {
      loadData()
    })
}
PATCH - 修改数据；根据id或uid

由事件对象获取结构中绑定的数据

function mod(id, rank) {
  axios.patch('http://127.0.0.1:5173/software/' + id, {
    rank: rank + 1
  })
    .then(res => {
      //更新完毕后，创新加载数据
      loadData()
    })
}
DELETE - 删除数据；根据id或uid
function del(id) {
  axios.delete('http://127.0.0.1:5173/software/' + id)
    .then(res => {
      loadData()
    })
}

根据用户id获取 - 用户提交前可以根据结果决定是不是已经提交；如果查询结果不为0，则已经提交，不允许再次提交

url: 'http://127.0.0.1:3000/user?id='+this.data.id
POST - 提交数据；json-server会自动增加id，类型字符串
wx.request({
  url: 'http://127.0.0.1:3000/user',
  method:'POST',
  data:{
    id:2,
    uname:'nj'
  },
  success:res=>{
    console.log(res);
  }
})
