自定义模块

@Module
协作、共享、解耦
每个模块都是一个js文件
区分系统模块和自定义模块
需要使用时，require进来；系统模块直接引入；自定义模块使用相对路径指定
建议集中管理自定义模块，如统一存放在module文件夹
基本步骤
创建模块文件
定制业务逻辑
导出模块
用户引入模块
用户使用模块
自定义模块 - 单个逻辑
以创建admin模块为例；具体过程如下：
在服务器的模块目录创建 admin.js
编辑内容并导出
const admin = {
  name: 'glpla',
  url: 'https://glpla.github.io',
  email: '1942194789@qq.com',
  date: '2024-02-28',
  version: '1.0.0',
  desc: 'for drill'
}
// 暴露1个
module.exports = admin
用户在自己的js文件引入并使用
const admin = require('./module/admin')
console.log('admin:\t', admin);
自定义模块 - 多个逻辑
以创建oper模块为例；具体过程如下：
在服务器的模块目录创建 oper.js
编辑内容并导出；包括add和dec两个逻辑
const add = () => {
  console.log('hi, add');
}
const dec = () => {
  console.log('hi, dec');
}
// 暴露2个 - 对象方式
module.exports = {
  add,
  dec
};
用户在自己的js文件引入并使用
const oper = require('./module/oper')
oper.add()
oper.dec()
解构引入并使用 - 更加灵活，需要哪个引入哪个
const { add,dec } = require('./module/oper')
add()
dec()
定制多个逻辑，解构化引入
引入时，可以省略后缀名.js
