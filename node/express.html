Express框架

@Express
req
请求
路由参数 - url/:id
server.get('/:id', (req, res) => {
  res.send(req.params.id)
})
res
响应
write() - 写回流时，需要显式指定结束 end
res.write(data)
res.end()
send() - 发送普通字符
res.send('hi,there.') 
res.send('<h1>hi,there.</h1>') 
res.send({ id: 10, name: 'gl' })
sendFile() - 需要指定参考路径
res.sendFile('./www/index.html', { root: __dirname })
json() - 返回json数据
res.json({ id: 10, name: 'gl' })
render() - 页面路径依据配置的静态资源路径
res.render('home/index', {
  navIndex: 0
})
其它
res.status(200).json({ msg: 'cnplaman', success: true })
404
以中间件use()的形式实现
所有路由都匹配失败，则匹配404；所以404中间件应该放在所有路由的 后面
//其它路由

server.use((req, res) => {
  res.status(404).sendFile('./www//lost.html', { root: __dirname })
})
中间件在req和res之间执行；拿到res后就没有必要了
