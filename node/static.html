静态资源服务器

@StaticServer
创建静态资源服务器，供应用程序使用，如在线图片
没有使用原生http模块，而是依托express框架创建；更多信息，请访问 Express.js
基本步骤
创建服务器目录
准备数据
创建服务器脚本
开启服务
具体过程
创建一个空白文件夹作为服务器目录，如server
在服务器目录创建资源文件夹，如www，存放数据data、图片imgs、视频video等资源；建议不同类型的资源存放在不同的子目录
在服务器目录打开CMD命令行窗口

初始化，用来创建package.json文件

npm init

也可以使用-y命令行参数，按照默认配置创建package.json，如入口文件默认是index.js - 推荐

npm init -y
{
  "name": "server",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}
安装express框架
npm i express --save
创建服务器脚本index.js - 引入express；创建服务器；指定静态文件位置；指定端口并监听
//引入express
const express = require('express');
//创建服务器实例
const app = express();
//指定服务器端口
const port = 3000
//指定静态文件位置
app.use(express.static('./www'));
//监听
app.listen(port, res => {
  console.log('服务器启动成功，访问地址：http://127.0.0.1:3000/资源名');
});
定制路由

根据业务场景和需求定制；如读取文件并返回JSON数据；如果如下提示，需要使用代理。见后续内容

...has been blocked by CORS policy...
开启服务

默认命令

node index.js

自定义命令启动服务 - 修改package.json里script

"scripts": {
  "start":"node index.js"
}
node start
使用资源

用户使用浏览器访问 http://127.0.0.1:3000/资源路径和名字获取资源

用户在H5项目中使用，如<img>元素

<img src="http://127.0.0.1:3000/资源路径和名字" alt="">

用户在微信小程序项目中使用，如<image>组件

<image src="http://127.0.0.1:3000/资源路径和名字" alt="">
日常开发使用Vs Code完成
进阶
访问服务器根目录，渲染服务器主页，给出操作提示
思路：创建html页面，指定路由；读取并返回主页；需要fs的支持；或直接使用res的sendFile()
修改服务器文件，增加核心代码如下
server.get('/', (req, res) => {
  fs.readFile('www/index.html', (err, data) => {
    if (err) throw err
    res.write(data)
    res.end()
  })
})
代理
解决跨域的冲突；注意调整接口的使用
修改项目vite.config.js中的服务器server配置，增加对访问接口的代理
server: {
  open: true,
  port: '3000',
  proxy: {
    '/store': {
      target: 'http://127.0.0.1:3000',
      changeOrigin: true,
      // rewrite: (path) => path.replace(/^\/store/, '') // 不可以省略rewrite
    }
  }
}
