静态资源服务器

@StaticServer
创建静态资源服务器，供应用程序使用，如在线图片、在线视频、在线音乐
没有使用原生http模块，而是依托express框架创建
更多信息，请访问 Express.js
基本步骤
创建服务器目录
准备数据
创建服务器脚本
开启服务
具体过程
创建一个空白文件夹作为服务器目录，如server
在服务器目录创建资源文件夹，如public，存放图片imgs、视频video、数据data等资源；建议不同类型的资源存放在不同的子目录
在服务器目录打开CMD命令行窗口：右键选择"在终端中打开"或在资源管理器顶部地址栏输入 CMD 并回车

初始化，用来创建 package.json 文件

npm init

推荐使用 -y 命令行参数，按照默认配置创建 package.json，如入口文件默认是index.js

npm init -y

创建的 package.json 文件内容如下

{
  "name": "server",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}
在CMD终端中安装 express 框架
npm i express --save
创建服务器脚本：在服务器目录下，新建一个js文件，命名为index.js，编辑如下
//引入express
const express = require('express');
//创建服务器实例
const app = express();
//指定服务器端口
const port = 3000
//指定静态文件位置
app.use(express.static('./public'));
//监听
app.listen(port, res => {
  console.log('服务器启动成功，访问地址：http://127.0.0.1:3000/资源名');
});
定制路由[可选]

根据业务场景和需求定制；如读取文件并返回JSON数据；如果如下提示，需要使用代理。见后续内容

...has been blocked by CORS policy...
在CMD终端中开启服务

默认命令

node index.js

自定义命令启动服务：修改package.json里script

"scripts": {
  "start":"node index.js"
}
node start
使用资源

用户使用浏览器访问 http://127.0.0.1:3000/资源路径和名字获取资源；如图片的src属性

用户在H5项目中使用，如<img>元素

<img src="http://127.0.0.1:3000/资源路径和名字" alt="">

用户在微信小程序项目中使用，如<image>组件

<image src="http://127.0.0.1:3000/资源路径和名字" alt="">
日常开发建议使用Vs Code完成
进阶
访问服务器根目录，渲染服务器特定页面，如主页，给出操作提示
思路：创建html页面，指定路由；读取并返回主页；需要fs的支持
方案1：修改服务器文件，增加核心代码如下
server.get('/', (req, res) => {
  fs.readFile('www/index.html', (err, data) => {
    if (err) throw err
    res.write(data)
    res.end()
  })
})
方案2：直接使用res的sendFile()
app.get('/', (req, res) => {
  res.sendFile('./www/index.html')
})
方案3：终极必杀技
指定静态资源，且有主页文件，且名字为index.html，即使没有定义路由也可以访问。。。
