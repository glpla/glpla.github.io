特性说明

浮动元素会脱离文档流document flow

布局时，元素的先后顺序不一样，最后呈现的布局也不一样

布局时，必须要清除浮动。你不清试试

现在：仅仅在一些特殊应用如：图文混排或首字下沉、根据形状浮动填充时，才会使用

其它布局请采用弹性盒子flex或网格grid；只推荐这俩种

清除浮动

清除的方法很多。如：为父元素指定高度；父元素溢出隐藏；

为了不增加额外的标签，通常以伪元素::after的形式为浮动元素的父元素指定清除类

    .clear::after {
      content: '';
      display: block;
      clear: both;
    }
