仓库操作
克隆仓库 clone
将远程仓库复制到本地
通常是第一次使用别人的仓库；后续直接拉取pull就可以和仓库保持一致
git clone url
Git关联到仓库
使用add为远程仓库指定一个别名；因为仓库url太长
可以关联多个远程仓库，使用别名区分
示例中使用origin代替远程仓库的url
$git remote add origin url
查看关联的远程仓库 remote
$ git remote -v
origin  https://github.com/glpla/glpla.github.io.git (fetch)
origin  https://github.com/glpla/glpla.github.io.git (push)            
$ git remote
origin
删除远程仓库和别名的关联 remove
$ git remote remove 别名
重命名远程仓库别名 rename
$ git remote rename 旧别名 新别名
Git推送到仓库 push
不仅要指定推送的仓库/别名，还要指定推送到哪个分支
使用-u可以自动关联，后续只需要使用 git push 即可推送
示例中推送到远程仓库别名是 origin 的 master 分支
$git push -u 别名 分支
$git push
拉取远程仓库 pull
在第一次克隆 clone 的基础上，通过拉取和原来仓库保持一致
如果之前提交使用 -u，可以直接拉取
git pull 别名 分支
git pull
