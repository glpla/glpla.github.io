简介 Overview
文档流 Workflow
.渲染页面时，按照html元素的特性，从左到右、从上到下依次排列，形成文档流 - 常规流
.如果html元素是inline，则不换行排列；如果当前行放不下，就另起新行排列
.如果html元素是block，则独占1行，即使有多余的空间也不给其它元素使用
脱离文档流
.CSS某些属性会导致html元素脱离文档流，如浮动、定位等，不再参与文档流的排列，即：位置丢失
.脱离了文档流的html元素就漂在页面中，由特有的CSS属性确定其最终的位置
浮动 Float
.浮动元素会脱离文档流，漂到容器的左侧或者右侧，原先位置丢失；位置丢失会怎样？后面元素会补上来
.left：先靠上再靠左；right：先靠上再靠右
.横向无缝排列：利用这个特性可以实现布局，特别是自适应布局，现在逐渐被弹性盒子flex或网格布局grid代替
特点 Features
.元素浮动后，变为块元素block
.元素在文档中出现的顺序不一样，最后渲染的结果也不一样
.消除inline元素的缝隙，如图片的空白
.使得block元素不再独占一行
.float布局时，若有一个元素浮动，则所有元素都应该浮动
.更多信息，请查看 float、float何去何从
清除浮动 Clear
.元素浮动后，会脱离正常的文档流
.如果不给父级指定高度，那么父级高度就变成了0/位置丢失[高度坍塌]；父级后面的元素就会补上来，造成布局混乱。而且大多数情况下，父级的高度一开始并不能确定，通常由子元素撑满；所以必须要清除浮动带来的影响
.只有块block元素可以清除浮动
.清除的方法很多。如：增加标签、为父元素指定高度；父元素溢出隐藏、使用布局属性如flex、gird
.为了不增加额外的标签，通常以伪元素::after的形式为浮动元素的父元素指定清除类
.请不要考虑IE的兼容性了。否则打你！！！
